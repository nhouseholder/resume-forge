## 2. What Changed

- **Fix 1 ÔÇö Preconnect dedup (`src/portfolio/buildPortfolioHtml.ts`):** `fontUrls.map()` was emitting two `<link rel="preconnect">` tags per iteration ÔÇö one pair per font URL. Refactored to emit the preconnect pair once, then map only `<link rel="stylesheet">` per URL. Verified: iframe head contains exactly 1 preconnect pair.
- **Fix 2 ÔÇö Popup-block feedback (`src/components/export/htmlExporter.ts`, `src/components/export/ExportPanel.tsx`):** `printPortfolioPdf` previously silently returned `void` when `window.open()` was blocked. Changed to return `{ ok: true } | { ok: false; reason: 'popup-blocked' }`. ExportPanel now calls `setNotice({ kind: 'error', message: 'Popup blockedÔÇª' })` on failure. Verified: error notice appears.
- **Fix 3 ÔÇö IDB async hydration (`src/pages/BuilderPage.tsx`):** `useState<Step>(resume ? 'editor' : 'upload')` is evaluated at mount time before IndexedDB resolves. Added `useEffect` that watches `resume` and transitions step to `'editor'` when resume becomes non-null while step is still `'upload'`. This means navigating directly to `/builder` with a saved resume now correctly loads the editor. Verified: builder renders on fresh page load with injected IDB data.
- **Fix 4 ÔÇö themeResolver defensive fallback (`src/utils/themeResolver.ts`):** `DENSITY_SCALE[meta.layoutDensity]` returned `undefined` when `layoutDensity` was missing from stored meta (e.g., old saved data). Added `?? DENSITY_SCALE.balanced` fallback matching the existing defensive pattern on lines 76ÔÇô77 for palette/font.
- **Fix 5 ÔÇö TemplateRenderer defensive arrays (`src/components/templates/TemplateRenderer.tsx`):** `applySectionVisibility` passed `data.publications` (and 13 other optional sections) through when visible ÔÇö returning `undefined` when the field was absent. Changed all 14 array sections to `data.field ?? []` to guarantee arrays. Also added `?.` to `meta.sectionVisibility[key]` to guard against undefined `sectionVisibility`.
